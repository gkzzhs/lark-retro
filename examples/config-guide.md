# lark-retro 配置指南

`lark-retro` 开箱即用，无需任何配置文件。以下是进阶用法的说明。

---

## 1. 权限配置

### 最小化权限（基础回顾）

只需日历和任务权限，即可生成包含时间分配和任务完成率的基础回顾报告：

```bash
lark-cli auth login --domain calendar,task
```

### 完整权限（全功能回顾）

授权所有相关业务域，解锁消息分析、文档搜索、知识库归档和群聊通知：

```bash
lark-cli auth login --domain calendar,task,doc,im,wiki
```

### 检查当前权限

```bash
lark-cli auth check
```

---

## 2. 知识库归档配置

要将回顾报告归档到知识库，你需要告诉 AI 两个信息：

### 获取知识库空间 ID

```bash
# 列出你有权限的知识库空间
lark-cli wiki +list
```

从输出中找到目标空间的 `space_id`。

### 获取父节点 Token

在飞书知识库中，找到你想要存放回顾报告的父节点（比如"团队回顾"文件夹），从 URL 中获取 `node_token`。

### 使用示例

```
帮我做上周的回顾，存到知识库空间 xxx 的节点 xxx 下面
```

> 首次使用后，AI 会记住你指定的知识库位置。

---

## 3. 群聊通知配置

要在回顾完成后自动通知团队，你需要提供群聊 ID：

### 获取群聊 ID

```bash
# 搜索群聊
lark-cli im +chats-search --query "你的群名"
```

从输出中找到 `chat_id`。

### 使用示例

```
做完回顾后发到群聊 oc_xxx 里
```

---

## 4. 自定义回顾周期

默认回顾周期为过去 7 天。你可以用自然语言指定任意时间范围：

| 说法 | 效果 |
|------|------|
| "上周的回顾" | 上周一 ~ 上周日 |
| "这周的回顾" | 本周一 ~ 现在 |
| "过去两周" | 14 天前 ~ 现在 |
| "三月份的回顾" | 3月1日 ~ 3月31日 |
| "上个 sprint（两周一迭代）" | 28天前 ~ 14天前 |

## 5. 渐进增强特性

| 功能 | 所需授权 | 效果 |
|------|---------|------|
| 日程分析 | `calendar` | 会议分类、时间分配、密度分析 |
| 任务追踪 | `task` | 完成率、延期分析、Blocker 识别 |
| 消息分析 | `im` | 关键讨论、决策提取、风险预警 |
| 文档上下文 | `doc` | 上期报告对比、趋势分析 |
| 知识库归档 | `wiki` | 报告沉淀、团队知识管理 |
| 群聊通知 | `im` | 自动推送报告摘要给团队 |

每个功能模块独立运作——缺少某个授权时，对应模块跳过，不影响其他功能。

---

## 6. 常见问题

**Q: 回顾报告里的数据准确吗？**

A: 所有数据直接来自飞书 API，数值（会议数、任务完成率等）是精确的。AI 生成的分析和建议基于这些数据，但属于推断性内容，建议团队讨论确认。

**Q: 可以给整个团队做回顾吗？**

A: 当前版本基于个人视角（你的日历、你的任务）。团队级回顾需要每个成员授权，是未来可扩展的方向。

**Q: 上期回顾报告怎么找到的？**

A: AI 通过 `docs +search` 搜索标题含"回顾"或"retrospective"的文档。如果你把回顾报告归档到知识库的固定节点下，查找会更准确。

**Q: 行动项创建为任务后在哪里看？**

A: 在飞书任务中心查看。AI 创建任务前会先列出清单让你确认，不会静默创建。
