# lark-retro

**AI 驱动的 Sprint 回顾引擎** — 一句话触发完整回顾流程：自动采集数据、分析模式、生成结构化报告、沉淀到知识库、追踪改进项闭环。

[English](README.md)

---

## 痛点

Sprint 回顾（Retrospective）是团队最有价值的仪式之一，但现实中往往：

- **流于表面**：上周发生了什么？大家记不清，反馈只能泛泛而谈
- **数据分散**：日程、消息、任务、文档各自为政，没人去做交叉分析
- **改进项遗忘**：上次回顾的 Action Items？早就忘了
- **浪费时间**：60 分钟的会议，真正有洞察的部分可能只有 10 分钟

## 解决方案

`lark-retro` 把回顾从一场会议变成一个 **数据驱动的自动化工作流**：

```
你说："帮我做一下上周的回顾"
```

AI Agent 自动完成：

1. **数据采集** — 从日历、任务、消息、文档 4 个飞书领域拉取工作数据
2. **模式分析** — 计算时间分配、任务完成率、识别 Blocker 和关键决策
3. **报告生成** — 输出结构化回顾（做得好的 / 待改进的 / 行动项 / 趋势对比）
4. **文档沉淀** — 自动创建飞书文档，可选挂载到知识库
5. **任务创建** — 每个行动项自动创建飞书任务，带负责人和截止日期
6. **闭环追踪** — 下次回顾时自动检查上期行动项是否落地

```
│  ┌─── 数据采集层 ───────────────────────────────────┐   │
│  │ calendar +agenda         → 时间分配分析          │   │
│  │ task +get-my-tasks       → 任务完成率            │   │
│  │ im +messages-search      → Blocker & 关键决策    │   │
│  │ docs +search             → 上期回顾上下文         │   │
│  └──────────────────────────┬───────────────────────┘   │
│                             │                           │
│  ┌─── AI 分析层 ────────────▼───────────────────────┐   │
│  │ · 会议分类（1:1 / 评审 / 站会 / 其他）           │   │
│  │ · 完成率计算 & 趋势对比                          │   │
│  │ · 消息中提取 Blocker                             │   │
│  │ · 与上期回顾对比                                 │   │
│  └──────────────────────────┬───────────────────────┘   │
│                             │                           │
│  ┌─── 输出层 ───────────────▼───────────────────────┐   │
│  │ docs +create       → 回顾文档                    │   │
│  │ docs +create       → 知识库归档（--wiki-space）   │   │
│  │ task +create       → 行动项任务                   │   │
│  │ im +messages-send  → 群聊通知                    │   │
│  └──────────────────────────────────────────────────┘   │
│                                                         │
│  ┌─── 闭环层 ───────────────────────────────────────┐   │
│  │ 下次回顾 → 自动追踪上期行动项完成情况             │   │
│  └──────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

## 有什么不同

| 维度 | 现有官方 Skill | lark-retro |
|------|---------------|------------|
| **范围** | 单一领域操作 | 跨 5 个领域编排 |
| **智能度** | 执行命令 | 分析数据、发现规律、生成洞察 |
| **连续性** | 单次操作 | 跨周期闭环追踪 |
| **输出** | 原始数据或简单摘要 | 结构化报告 + 文档归档 + 任务创建 + 群聊通知 |

## 安装

### 前置要求

- Node.js >= 18
- [lark-cli](https://github.com/larksuite/cli) 已安装并配置

### 安装步骤

```bash
# 1. 安装 lark-cli（如尚未安装）
npm install -g @larksuite/cli

# 2. 安装官方 Skills（包含 lark-shared 等基础依赖）
npx skills add https://github.com/larksuite/cli -y -g

# 3. 安装 lark-retro
npx skills add https://github.com/gkzzhs/lark-retro -y -g

# 4. 配置并登录（核心功能）
lark-cli config init --new
lark-cli auth login --domain calendar,task,docs

# 可选：启用消息搜索和文档搜索（增强功能）
lark-cli auth login --scope "search:message search:docs:read"

# 5. 重启你的 AI Agent 工具（Trae / Cursor / Claude Code / Codex）
```

## 使用示例

### 基础回顾（日历 + 任务）

```
帮我做一下上周的回顾
```

### 完整回顾（含消息分析）

```
帮我复盘一下过去两周的工作，包括群聊里的关键讨论
```

### 回顾 + 知识库归档

```
生成这个 Sprint 的回顾报告，存到知识库的"团队回顾"节点下
```

### 追踪上期行动项

```
上周回顾里的行动项完成了吗？顺便做一下这周的回顾
```

### 工作周报生成

```
帮我写这周的周报，基于日历和任务数据
```

## 所需权限

| 功能 | 授权方式 | 是否必须 |
|------|---------|----------|
| 日程分析 | `--domain calendar` | 是 |
| 任务追踪 | `--domain task` | 是 |
| 报告创建 | `--domain docs` | 是 |
| 消息分析 | `--scope "search:message"` | 可选 |
| 文档搜索 | `--scope "search:docs:read"` | 可选 |
| 群聊通知 | Bot 身份（需在开发者后台开通） | 可选 |

## 技术特点

- **零代码，纯 Skill**：完全通过 `SKILL.md` 实现，无脚本、无二进制文件、无外部依赖
- **100% lark-cli 原生**：所有操作使用内置命令
- **渐进增强**：核心功能（日历+任务）只需最少权限；高级功能（消息分析、知识库归档）按需开启
